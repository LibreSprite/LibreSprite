<p>To compile LibeSprite you will need:</p>
<ul>
<li>The latest version of <a href="http://www.cmake.org/" rel="nofollow">CMake</a> (3.4 or greater)</li>
<li><a href="https://ninja-build.org" rel="nofollow">Ninja</a> build system</li>
</ul>
<p>LibreSprite can be compiled with two different back-ends:</p>
<ol>
<li>
<p>Allegro back-end (Windows, Linux): You will not need any extra
library because the repository already contains a modified version
of the Allegro library. This back-end is deprecated and will be
removed in future versions. All new development is being done in
the new Skia back-end.</p>
</li>
<li>
<p>Skia back-end (Windows, Mac OS X): You will need a compiled version
of the Skia library. Please check the details about
<a href="#building-skia-dependency">how to build Skia</a> on your platform.</p>
</li>
</ol>

<p>First of all, you will need an extra little utility: <code>awk</code>, used to
compile the libpng library. You can get this utility from MSYS2
distributions like <a href="https://wiki.mozilla.org/MozillaBuild" rel="nofollow">MozillaBuild</a>.</p>
<p>After that you have to choose the back-end:</p>
<ol>
<li>
<p>If you choose the Allegro back-end, you can jump directly to the
<a href="#compiling">Compiling</a> section.</p>
</li>
<li>
<p>If you choose the Skia back-end, you will need to
<a href="#skia-on-windows">compile Skia</a> before and then continue in the
<a href="#compiling">Compiling</a> section. Remember to check the
<a href="#windows-details">Windows details</a> section to know how to call
<code>cmake</code> correctly.</p>
</li>
</ol>

<ol>
<li>
<p><a href="#get-the-source-code">Get LibreSprite code</a>, put it in a folder like
<code>C:\LibreSprite</code>, and create a <code>build</code> directory inside to leave all
the files that are result of the compilation process (<code>.exe</code>,
<code>.lib</code>, <code>.obj</code>, <code>.a</code>, <code>.o</code>, etc).</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content=" cd C:\LibreSprite
 mkdir build
"><pre><code> cd C:\LibreSprite
 mkdir build
</code></pre></div>
<p>In this way, if you want to start with a fresh copy of LibreSprite
source code, you can remove the <code>build</code> directory and start again.</p>
</li>
<li>
<p>Enter in the new directory and execute <code>cmake</code>:</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content=" cd C:\LibreSprite\build
 cmake -G Ninja ..
"><pre><code> cd C:\LibreSprite\build
 cmake -G Ninja ..
</code></pre></div>
<p>Here <code>cmake</code> needs different options depending on your
platform. You must check the details for
<a href="#windows-details">Windows</a>, <a href="#mac-os-x-details">OS X</a>, and
<a href="#linux-details">Linux</a>. Some <code>cmake</code> options can be modified using tools like
<a href="https://cmake.org/cmake/help/latest/manual/ccmake.1.html" rel="nofollow"><code>ccmake</code></a>
or <a href="https://cmake.org/cmake/help/latest/manual/cmake-gui.1.html" rel="nofollow"><code>cmake-gui</code></a>.</p>
</li>
<li>
<p>After you have executed and configured <code>cmake</code>, you have to compile
the project:</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content=" cd C:\LibreSprite\build
 ninja libresprite
"><pre><code> cd C:\LibreSprite\build
 ninja libresprite
</code></pre></div>
</li>
<li>
<p>When <code>ninja</code> finishes the compilation, you can find the executable
inside <code>C:\LibreSprite\build\bin\libresprite.exe</code>.</p>
</li>
</ol>

<p>If you don't want to use the embedded code of third party libraries
(i.e. to use your installed versions), you can disable static linking
configuring each <code>USE_SHARED_</code> option.</p>
<p>After running <code>cmake -G</code>, you can edit <code>build/CMakeCache.txt</code> file,
and enable the <code>USE_SHARED_</code> flag (set its value to <code>ON</code>) of the
library that you want to be linked dynamically.</p>
<h2><a id="user-content-linux-issues" class="anchor" aria-hidden="true" href="#linux-issues"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Linux issues</h2>
<p>If you use the official version of Allegro 4.4 library (i.e. you
compile with <code>USE_SHARED_ALLEGRO4=ON</code>) you will experience a couple of
known issues solved in
<a href="https://github.com/LibreSprite/LibreSprite/tree/master/src/allegro">our patched version of Allegro 4.4 library</a>:</p>
<ul>
<li>You will
<a href="https://github.com/aseprite/aseprite/issues/192">not be able to resize the window</a>
(<a href="https://github.com/LibreSprite/LibreSprite/commit/920f6275d55113507121afcbcda80adb44cc0563">patch</a>).</li>
<li>You will have problems
<a href="https://github.com/LibreSprite/LibreSprite/commit/27b55030e26e93c5e8d9e7e21206c8709d46ff22">adding HSV colors in non-English systems</a>
using the warning icon.</li>
</ul>
<h1><a id="user-content-building-skia-dependency" class="anchor" aria-hidden="true" href="#building-skia-dependency"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Building Skia dependency</h1>
<p>When you compile LibreSprite with <a href="https://skia.org" rel="nofollow">Skia</a> as back-end on
Windows or OS X, you need to compile a specific version of Skia. In
the following sections you will find straightforward steps to compile
Skia.</p>
<p>You can always check the
<a href="https://skia.org/user/quick" rel="nofollow">official Skia instructions</a> and select
the OS you are building for. LibreSprite uses the <code>aseprite-m53</code> Skia
branch from <code>https://github.com/aseprite/skia</code>.</p>
<h2><a id="user-content-skia-on-windows" class="anchor" aria-hidden="true" href="#skia-on-windows"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Skia on Windows</h2>
<p>Download
<a href="https://storage.googleapis.com/chrome-infra/depot_tools.zip" rel="nofollow">Google depot tools</a>
and uncompress it in some place like <code>C:\deps\depot_tools</code>.</p>
<p>Then open a command line follow these steps (for VS2015):</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="call &quot;%VS140COMNTOOLS%\vsvars32.bat&quot;

"><pre><code>call "%VS140COMNTOOLS%\vsvars32.bat"
set PATH=C:\deps\depot_tools;%PATH%

gclient sync
</code></pre></div>
<p>(The <code>gclient</code> command might print an error like
<code>Error: client not configured; see 'gclient config'</code>.
Just ignore it.)</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="cd C:\deps
git clone https://github.com/aseprite/skia.git
cd skia
git checkout aseprite-m53

"><pre><code>cd C:\deps
git clone https://github.com/aseprite/skia.git
cd skia
git checkout aseprite-m53
python bin/sync-and-gyp
</code></pre></div>
<p>(The <code>bin/sync-and-gyp</code> will take some minutes because it downloads a
lot of packages, please wait and re-run the same command in case it fails.)</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="ninja -C out/Release dm
"><pre><code>ninja -C out/Release dm
</code></pre></div>
<p>More information about these steps in the
<a href="https://skia.org/user/quick/windows" rel="nofollow">official Skia documentation</a>.</p>
<h2><a id="user-content-skia-on-mac-os-x" class="anchor" aria-hidden="true" href="#skia-on-mac-os-x"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Skia on Mac OS X</h2>
<p>These steps will create a <code>deps</code> folder in your home directory with a
couple of subdirectories needed to build Skia (you can change the
<code>$HOME/deps</code> with other directory). Some of these commands will take
several minutes to finish:</p>
<div class="snippet-clipboard-content position-relative" data-snippet-clipboard-copy-content="mkdir $HOME/deps

<p>More information about these steps in the
<a href="https://skia.org/user/quick/macos" rel="nofollow">official Skia documentation</a>.</p>
